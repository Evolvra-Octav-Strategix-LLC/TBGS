**b) ISR Revalidate API (Vercel)**

```ts
// app/api/revalidate/route.ts
import { NextRequest, NextResponse } from 'next/server';

export async function POST(req: NextRequest) {
  const secret = req.headers.get('x-webhook-secret');
  if (secret !== process.env.REVALIDATE_SECRET) {
    return NextResponse.json({ ok: false }, { status: 401 });
  }

  const { paths } = await req.json(); // array: ['/nl/locaties/eindhoven', '/be/totaal-.../lommel']
  try {
    const revalidated: string[] = [];
    for (const p of paths) {
      // @ts-ignore
      await (global as any).revalidatePath?.(p) // Next 14 app router helper in route handlers
      revalidated.push(p);
    }
    return NextResponse.json({ ok: true, revalidated });
  } catch (e) {
    return NextResponse.json({ ok: false, error: String(e) }, { status: 500 });
  }
}
```

# üîé Wanneer Webflow of Drupal i.p.v. Headless?

## Webflow (wanneer w√©l)

* Je wil **supersnel live** met **pixel‚Äëperfect design** en **weinig techniek**.
* Je blijft **<10.000 CMS items** (TBGS past hier meestal in).
* Nadeel: minder ‚Äúcode‚Äëfirst‚Äù vrijheid dan bovenstaande hybride.

**TBGS‚Äëadvies:** Webflow is top als je *geen* headless wilt en t√≥ch strak en snel. Voor jouw **hybride wens (tekst ook in code)** is **Next.js + WP** flexibeler.

## Drupal (wanneer w√©l)

* **Enterprise**: honderden redacteuren, extreme permissions, multi‚Äësite, custom workflows.
* Zwaarder beheer, langere doorlooptijd. Overkill voor TBGS tenzij je echt franchise/backoffice op schaal wilt.

---

## TL;DR ‚Äì Mijn aanbeveling

* **Kies de hybride**: **Next.js + headless WordPress** (zoals hierboven).
  ‚Üí Snel, SEO‚Äëmax, schaalbaar, √©n jij kunt teksten zowel in WP als in code aanpassen.
* **Alternatief**: Webflow als je zero‚Äëbackend wilt.
* **Alleen Drupal** als je enterprise‚Äëredactie en complexe workflows nodig hebt.
